    HTTP Request

    POST https://testnet.sectoken.io/api/v1/wallet/fil/create/

    请求参数

    参数名 参数类型 必填 说明
    pubkey string 是 BLS公钥，长度为96
    privkey string 否 私钥，用户应自行加密传输，使用时解密。（必须使用base64编码）
    wallet_name string 是 钱包名

    返回参数

    参数名 参数类型 说明
    code integer 返回码，200成功，其他失败，请查看错误码
    data object 返回内容

    data 参数

    参数名 参数类型 说明
    wallet_id string 钱包的id
    activate_code string 钱包激活码

    HTTP Request

    POST https://testnet.sectoken.io/api/v1/wallet/fil/share_create/

    请求参数

    参数名 参数类型 必填 说明
    pubkey array 是 BLS公钥，长度为96
    privkey array 否 私钥, 必须为Base64格式
    wallet_name string 是 钱包名称
    threshold integer 是 钱包最少签名个数,最大为7, 必须和limit一致
    limit integer 是 钱包公钥总个数,最大为7, 必须和threshold一致
    control integer 否 0:平台与用户都需要签名; 1:仅用户签名, 默认为 0
    multi integer 否 0:用户只能持有一个公钥; 1:用户允许持有多个公钥, 默认为0

    返回参数

    参数名 参数类型 说明
    code integer 返回码，200成功，其他失败，请查看错误码
    data object 返回内容

    data 参数

    参数名 参数类型 说明
    invitation_code string 分享码，其他用户输入此码可加入此多用户钱包

    tx_str string 交易单信息
    gas_price integer 交易所花费的 gas 单价
    gas_limit integer 交易所花费的 gas 上限

    交易Webhook通知

    当用户钱包发生交易时，会向指定的url发送HTTP POST请求，请求头content-type为application/json。

    此url由用户指定，用户须部署服务接收HTTP POST请求。同时需先添加钱包Webhook。

    请求头中会有Checksum参数，用于验证请求信息是否完整， 通过 sha256（secret_key+request.body）来进行哈希.

    secret_key 为创建webhook时提交的secret_key, 如果创建时没有上传,默认为空字符串.

    请求headers示例

    {
  "Content_type": "application/json",
  "Checksum": "sha256=6ca13d52ca70c883e0f0bb101e425a89e8624de51db2d2392593af6a84118090"
}
