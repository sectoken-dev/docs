    获取当前账户的费用地址

    import requests

requests.get(
    url="https://testnet.sectoken.io/api/v1/wallet/eth/fee/", 
    headers={"HMAC": "your token"}
)

    curl "https://testnet.sectoken.io/api/v1/wallet/eth/fee/" \
    -X GET \
    -H "HMAC:your token"

    let request = require("request");

request({
    url: "https://testnet.sectoken.io/api/v1/wallet/eth/fee/",
    method: "get",
    headers: {
        "Content-Type": "application/json",
        "HMAC": "your token",
    },
});

    返回示例

    {
    "code": 200,
    "data": {
      "address": "0x3A857010Bd2CdEDF6d0a6C90aBe7C50f34038a62",
      "balance": "318062988000000000"
    }
}

    获取当前账户的费用地址，如果不存在，则创建新的一个地址。费用地址作用：

    用来支付创建多签钱包部署合约的矿工费

    用来支付新增收款地址合约部署的的矿工费

    用来支付多签钱包转账的矿工费

    把ERC20从钱包收款地址归集到钱包多签地址，也需要支付矿工费。

    费用地址建议充值0.01币以上。

    HTTP Request

    GET https://testnet.sectoken.io/api/v1/wallet/eth/fee/

    无请求参数

    返回参数

    参数名 参数类型 说明
    code integer 返回码，200成功，其他失败，请查看错误码
    data object 返回内容

    Data 参数

    参数名 参数类型 说明
    address string 费用地址
    balance string 余额，单位为Wei

    HTTP Request

    POST https://testnet.sectoken.io/api/v1/wallet/eth/share_create/

    请求参数

    参数名 参数类型 必填 说明
    pubkey array 是 公钥
    privkey array 否 私钥, 必须为Base64格式
    wallet_name string 是 钱包名称
    threshold integer 是 钱包最少签名个数,最大为7
    limit integer 是 钱包公钥总个数,最大为7
    control integer 否 0:平台与用户都需要签名; 1:仅用户签名, 默认为 0
    multi integer 否 0:用户只能持有一个公钥; 1:用户允许持有多个公钥, 默认为0

    返回参数

    参数名 参数类型 说明
    code integer 返回码，200成功，其他失败，请查看错误码
    data object 返回内容

    Data 参数

    参数名 参数类型 说明
    invitation_code string 分享码，其他用户输入此码可加入此多用户钱包

    生成收款地址

    生成收款地址,所需成本大约 320000 gas

    该地址收到eth后,自动转账到归属钱包的多签地址

    该地址收到ERC20代币后,需要手动触发归集转账到归属钱包的多签地址

    该地址不能对外部的任何地址进行支出转账

    import requests
requests.post(
    url="https://testnet.sectoken.io/api/v1/wallet/eth/derive/",
    headers={"HMAC": "your token"},
    json={"wallet_id": "cf761ad1b6c5f30a1557d942cb557b07"}
)

    curl "https://testnet.sectoken.io/api/v1/wallet/eth/derive/" \
    -X POST \
    -H "Content-Type:application/json" \
    -H "HMAC:your token" \
    -d '{"wallet_id": "cf761ad1b6c5f30a1557d942cb557b07"}'

    let request = require("request");

request({
        url: "https://testnet.sectoken.io/api/v1/wallet/eth/derive/",

    交易Callback通知

    当用户钱包发生交易时，会向指定的url发送HTTP POST请求，请求头content-type为application/json。

    此url由用户指定，用户须部署服务接收HTTP POST请求。同时需先添加钱包Webhook。

    请求头中会有Checksum参数，用于验证请求信息是否完整， 通过 sha256（secret_key+request.body）来进行哈希.

    secret_key 为创建webhook时提交的secret_key, 如果创建时没有上传,默认为空字符串.

    请求headers示例 json { "Content_type": "application/json", "Checksum": "sha256=6ca13d52ca70c883e0f0bb101e425a89e8624de51db2d2392593af6a84118090" }

    "height": 4340000,
