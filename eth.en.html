    Get Fee Address

    import requests

requests.get(
    url="https://testnet.sectoken.io/wallet/eth/fee/", 
    headers={"HMAC": "your token"}
)

    curl "https://testnet.sectoken.io/wallet/eth/fee/" \
    -X GET \
    -H "HMAC:your token" \

    Response Samples

    {
    "code": 200,
    "data": {
      "address": "0x1B004A73c74aC24891a1618e8b66C7dC95716E4b",
      "balance": 318062988000000000
    }
}

    Get fee address of the current account. Create if none exists.

    HTTP Request

    GET https://testnet.sectoken.io/wallet/eth/fee/

    No Request Parameters

    Response Parameters

    Parameter Name Type Description
    code integer Status code. Successful return 200. Others are error codes.
    data object Response data.

    Data Parameters

    Parameter Name Type Description
    address string Fee address.
    balance string Balance of fee address. (Uint is Wei)

    Create Multi-sign Wallet

    import requests

requests.post(
    url="https://testnet.sectoken.io/wallet/eth/getkey/", 
    headers={"HMAC": "your token"}, 
    json={
        "pubkeys": ["compressed pubkey1",...],
        "wallet_name": "your wallet name",
        "multi_type": "2-3",
        "order_type": 0,
    }
)

    curl "https://testnet.sectoken.io/wallet/eth/getkey/" \
    -H "HMAC:your token" \
    -H "Content-Type:application/json" \
    -X POST \
    -d '{"pubkeys": ["compressed pubkey1",...], \
        "wallet_name": "your wallet name", \
        "multi_type": "2-3", \
        "order_type": 0}'

    Response Samples

    {
    "code": 200,
    "data": {
        "wallet_id": "cf761ad1b6c5f30a1557d942cb557b07",
        "create_time": "2019-10-10T10:13:03.195",
        "active_code": "712968",
        "server_pubkey": "xpub6AZMDpGgTSsnXRfapb7wm3fMppz38ayMmHePm7A27noBxiiBV7YzPqX71dZMgHkRQZWSC2WK2dXQs6tGBqqXvo3TTUWW6BmSNucv5iVRDfa"
    }
}

    HTTP Request

    POST https://testnet.sectoken.io/wallet/eth/getkey/

    Request Parameters

    Parameter Name Type Required Description
    pubkeys array Yes Compressed public keys
    wallet_name string Yes Wallet name
    multi_type string Yes Multiple signature type. eg: "2-3", "3-5"
    order_type integer Yes Whether the platform is signed or not. 0: Yes, 1: No
    If the platform participates in the signature, the number of public keys to be provided must be equal to the total signature number -1. If the platform does not participate in the signature, the number of public keys required must be equal to the total number of signatures.

    Response Parameters

    Parameter Name Type Description
    code integer Status code. Successful return 200. Others are error codes.
    data object Response data.

    Data Parameters

    Parameter Name Type Description
    wallet_id string Wallet id
    create_time string Creation time of wallet
    activate_code string Activate code
    server_pub_key string Xpublic key

    -X GET    

    {
    "code": 200,
    "data": {
        "wallet_id": "2cc5abb2b9932ca1ec5513ad1a9685f9",
        "wallet_name": "TestSeven",

    HTTP Request

    GET https://testnet.sectoken.io/wallet/eth/activate/

    Request Parameters

    Parameter Name Type Required Description
    wallet_id string Yes Wallet id
    coin_type string No Coin type. ETH or other ERC20 token supported by the platform. Default is "ETH".

    Response Parameters

    Parameter Name Type Description
    code integer Status code. Successful return 200. Others are error codes.
    data object Response data.

    Data Parameters

    Parameter Name Type Description
    wallet_id string Wallet id
    wallet_name string Wallet name
    address string ETH address of wallet
    balance integer Balance. (Minimum unit)
    send_max integer Maximum amount per transfer. 0 is unlimited.
    send_day_limit integer Maximum number of transfers per day. 0 is unlimited.
    send_day_max integer Maximum number of transfers per day. 0 is unlimited.
    freeze_overtime integer Wallet freeze deadline. 0 is not frozen.

    Submit Signature

    Submit the signature result for the transaction id, and if the signature number is valid and reaches the broadcast limit, the transaction will be broadcast to the blockchain

    import requests

requests.post(
    url="https://testnet.sectoken.io/wallet/eth/send/"
    headers={"HMAC": "your token"},
    json={
        "wallet_id": "your wallet id",
        "order_id": "order id",
        "signature": "signature data"
    }
)

    curl "https://testnet.sectoken.io/wallet/eth/send/" \
    -X POST \
    -H "Content-Type:application/json", \
    -H "HMAC:your token" \
    -d '{"wallet_id": "your wallet id", \
        "order_id": "order id", \
        "signature": "signature data"}'

    Response Samples

    {
    "code": 200,
    "data": {
        "is_sended": true
    }
}

    HTTP Request

    POST https://testnet.sectoken.io/wallet/eth/send/

    Request Parameters

    Parameter Name Type Required Description
    wallet_id string Yes Wallet id
    order_id string Transaction id. (not ETH transaction id)
    signature string Yes Signature

    Response Parameters

    Parameter Name Type Description
    code integer Status code. Successful return 200. Others are error codes.
    data object Response data.

    Data Parameters

    Parameter Name Type Description
    is_sended bool Whether to broadcast.If the number of signatures does not reach the broadcast limit, it indicates that the final signature was not successful. Otherwise, it returns true.

    HTTP Request

    POST https://testnet.sectoken.io/wallet/eth/authority/

    Request Parameters

    Parameter Name Type Required Description
    wallet_id string Yes Wallet id
    user_email string Yes The Email of the user to be operated on
    operation string Yes Operation type. add/ remove/ change
    role string No New role. observer or operator. if operation is "remove", type is not required.

    Response Parameters

    Parameter Name Type Description
    code integer Status code. Successful return 200. Others are error codes.

    Error Codes

    The error code returned when an error occurs

    Error code Description
    400 Invalid params
    500 Unknown error
    4203 Insufficient Balance
    4204 Wallet frozen
    4302 The amount of a single transaction exceeds the limit
    4303 The trading amount exceeded the upper limit on the day
    4304 The number of trades exceeded the limit
    4305 The destination address is not on the white list
    4401 Authentication failed
    4403 The number of wallets reached the limit
    4404 No operation permissions
    4507 Token doesn't exist
    4509 Signature error
    4510 Duplicate signature
    4601 The number of derived addresses reaches an upper limit
    4602 There is no fee address for the current account
    4603 Can't transfer it to yourself
