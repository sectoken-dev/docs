    Introduction

    Tools that should be deployed on the intranet, Test case can use internal.sectoken.io

    ETH Multi-sign

    import requests 

requests.post("https://internal.sectoken.io/eth/signFirst", 
    json={
        "data": "ae1111", 
        "prikey": "98361e9e89320d7911153ddc23db573fcedc12da8df710af30d3977ecea3f1d1", 
        "message": True
    }
)

    curl "https://internal.sectoken.io/eth/signFirst" \
    -X POST \
    -H "Content-type: application/json" \
    -d '{"data": "ae1111", "prikey": "98361e9e89320d7911153ddc23db573fcedc12da8df710af30d3977ecea3f1d1", "message":true}'

    Response Samples

    {
"code":200,
    "data":{
        "signature":"042b6770744ecde764fdd4a8b0f4e9cdc10bf26c3894efaeeb59f2755225df962f91bf96714c59db5207c9dbd36aa93593d15747cb9d2888aa4eaea5aa7b4ee400",
    }
}

    HTTP Request

    POST https://internal.sectoken.io/eth/signFirst

    Request Parameters

    Parameter Name Type Required Description
    data string Yes Unsigned message
    prikey string Yes Private key
    message bool Yes Sign message or transaction. If multi-sign, use true.

    Response Parameters

    Parameter Name Type Description
    code integer Status code. Successful return 200. Parameter error return 401.
    data object Response data.

    Data Parameters

    Parameter Name Type Description
    signature string Message signature
