    import requests

requests.post(
        url="https://testnet.sectoken.io/api/v1/wallet/eth_single/create/",
        headers={"Authorization": "Bearer Atk38e9b38c64430d56ee34a05f485279ae"},
        json={
            "pubkey": "020fcbb44ff4a1f423499b1eb9b2bd9cbe467934467d4857451c7eab7f8cb2885b",
            "wallet_name": "TestWallet",
        }
)

    curl "https://testnet.sectoken.io/api/v1/wallet/eth_single/create/" \
    -H "Authorization: Bearer $BEARER_TOKEN" \
    -H "Content-Type:application/json" \
    -X POST \
    -d '{
        "pubkey": "020fcbb44ff4a1f423499b1eb9b2bd9cbe467934467d4857451c7eab7f8cb2885b",
        "wallet_name": "TestWallet"
    }'

    var request = require("request");

    },
    body: JSON.stringify({
        "pubkey": "020fcbb44ff4a1f423499b1eb9b2bd9cbe467934467d4857451c7eab7f8cb2885b",

    HTTP Request

    POST https://testnet.sectoken.io/api/v1/wallet/eth_single/create/

    请求参数

    参数名 参数类型 必填 说明
    pubkey string 是 压缩公钥，长度为66
    privkey string 否 私钥，用户应自行加密传输，使用时解密。（必须使用base64编码）
    wallet_name string 是 钱包名

    返回参数

    参数名 参数类型 说明
    code integer 返回码，200成功，其他失败，请查看错误码
    data object 返回内容

    data 参数

    参数名 参数类型 说明
    wallet_id string 钱包的id
    activate_code string 钱包激活码

    HTTP Request

    GET https://testnet.sectoken.io/api/v1/wallet/eth_single/detail/

    freeze_overtime integer 钱包冻结截止时间，0为未冻结

    HTTP Request

    POST https://testnet.sectoken.io/api/v1/wallet/eth_single/dotx/

    请求参数

    参数名 参数类型 必填 说明
    wallet_id string 是 钱包 id
    to_address string 是 收款地址
    value string 是 转账金额，最小单位
    coin string 否 币种，默认为 eth，可以指定为平台支持的 erc20 token，如 erc20_ht，erc20_usdt
    gas_price integer 否 gas 单价, 单位为wei
    total integer 否 全部转出，1 生效，此时 value 值不生效。

    返回参数

    参数名 参数类型 说明
    code integer 返回码，200成功，其他失败，请查看错误码
    data object 返回内容

    data 参数

    参数名 参数类型 说明
    unsigned_tx string 未签名的交易
    privkey sting 创建钱包时提交的privkey字段，未提交则返回空字符串
    sequence_id integer 订单号id
    gas_price integer 交易所花费的 gas 单价
    gas_limit integer 交易所花费的 gas 上限

    交易Webhook通知

    当用户钱包发生交易时，会向指定的url发送HTTP POST请求，请求头content-type为application/json。

    此url由用户指定，用户须部署服务接收HTTP POST请求。同时需先添加钱包Webhook。

    请求头中会有Checksum参数，用于验证请求信息是否完整， 通过 sha256（secret_key+request.body）来进行哈希.

    secret_key 为创建webhook时提交的secret_key, 如果创建时没有上传,默认为空字符串.

    请求headers示例

    {
  "Content_type": "application/json",
  "Checksum": "sha256=6ca13d52ca70c883e0f0bb101e425a89e8624de51db2d2392593af6a84118090"
}
