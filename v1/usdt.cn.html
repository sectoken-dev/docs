    获取当前账户的费用地址

    获取当前账户的费用地址，如果不存在，则创建新的BTC地址。费用地址作用：

    用来支付多签钱包转账的矿工费

    当转出钱包内全部的USDT的时候, 钱包内剩余的 BTC 余额将全部转入到 费用地址中

    建议费用地址充值 100000 聪

    import requests

requests.get(
    url="https://testnet.sectoken.io/api/v1/wallet/usdt/fee/", 
    headers={"HMAC": "your token"}
)

    curl "https://testnet.sectoken.io/api/v1/wallet/usdt/fee/" \
    -X GET \
    -H "HMAC:your token" \

    返回示例

    {
    "code": 200,
    "data": {
      "address": "2N91dXMR7p8ne2Asuy8vgkwZ4DtwVszeCDw",
      "confirmed": "10000000",
      "unconfirmed":"10000"
    }
}

    HTTP Request

    GET https://testnet.sectoken.io/api/v1/wallet/usdt/fee/

    无请求参数

    返回参数

    参数名 参数类型 说明
    code integer 返回码，200成功，其他失败，请查看错误码
    data object 返回内容

    data 参数

    参数名 参数类型 说明
    address string 费用地址
    confirmed string 已确认可以使用的BTC余额 (最小单位)
    unconfirmed string 未确认不可用的BTC余额 (最小单位)

    创建USDT多签钱包

    为当前用户快速创建一个 USDT 多签2-3钱包

    import requests

requests.post(
    url="https://testnet.sectoken.io/api/v1/wallet/usdt/create/", 
    headers={"HMAC": "your token"}, 
    json={
        "pubkeys": [
            "03a6725ab3589cb91b416706fb23fe3f0b179d9d46af95d36606dc03cc24b99ec2", 
           "0392bdf172b6aeba9d68c93d1adc22c10ea865caacf0f601a31e938e053b537845"
        ],
        "privkeys":[
            "your encrypted privkey 1", 
            "your encrypted privkey 2", 
        ],
        "wallet_name": "your wallet name",
    }
)

    curl "https://testnet.sectoken.io/api/v1/wallet/usdt/create/" \
    -H "HMAC:your token" \
    -H "Content-Type:application/json" \

        "pubkeys": [
            "03a6725ab3589cb91b416706fb23fe3f0b179d9d46af95d36606dc03cc24b99ec2", 
            "0392bdf172b6aeba9d68c93d1adc22c10ea865caacf0f601a31e938e053b537845"
        ], 
        "privkeys": [
            "your encrypted privkey 1", 
            "your encrypted privkey 2"
        ], 
        "wallet_name": "your wallet name"
    }'

    创建USDT多用户钱包

    通过该接口可以快速创建任意多签形式的USDT多用户钱包 其他用户可以通过此接口返回的 invitation_code 加入钱包

    import requests

requests.post(
    url="https://testnet.sectoken.io/api/v1/wallet/usdt/share_create/", 
    headers={"HMAC": "your token"}, 
    json={
            "pubkey":"02d654b7b6829df777a32267d366b4c24fd30fc1a61d816e043104edd198678e52", 
            "privkey": "your encrypted privkey",
            "threshold":2,
            "limit":2,
            "control": 1,
            "wallet_name": "USDTTestWallet",
            "multi":1
    }
)

    curl "https://testnet.sectoken.io/api/v1/wallet/usdt/share_create/"
    -H "HMAC:your token"
    -H "Content-Type:application/json"
    -X POST
    -d '{   
        "pubkey":"02d654b7b6829df777a32267d366b4c24fd30fc1a61d816e043104edd198678e52", 
        "privkey": "your encrypted privkey",
        "threshold":2,
        "limit":2,
        "control": 1,
        "wallet_name": "USDTTestWallet",
        "multi":1
    }'

    返回示例

    {
    "code": 200,
    "data": {
        "invitation_code": "cf761ad1b6c5f30a1557d942cb557b07"
    }
}

    HTTP Request

    POST https://testnet.sectoken.io/api/v1/wallet/usdt/share_create/

    请求参数

    参数名 参数类型 必填 说明
    pubkey array 是 公钥
    privkey array 否 私钥, 必须为base64格式
    wallet_name string 是 钱包名
    threshold integer 是 钱包最少签名个数,最大为7
    limit integer 是 钱包公钥总个数,最大为7
    control integer 是 0:平台参与签名; 1:平台不参与签名
    multi integer 是 0:每个用户只能使用一个公钥,1:每个用户可以使用多个公钥, 默认为0

    返回参数

    参数名 参数类型 说明
    code integer 返回码，200成功，其他失败，请查看错误码
    data object 返回内容

    data 参数

    参数名 参数类型 说明
    invitation_code string 分享码，其他用户输入此码可加入此多用户钱包

    交易Callback通知

    当用户钱包发生交易时，会向指定的url发送HTTP POST请求，请求头content-type为application/json。

    此url由用户指定，用户须部署服务接收HTTP POST请求。同时需先添加钱包Webhook。

    请求头中会有Checksum参数，用于验证请求信息是否完整， 通过 sha256（secret_key+request.body）来进行哈希.

    secret_key 为创建webhook时提交的secret_key, 如果创建时没有上传,默认为空字符串.

    请求headers示例 json { "Content_type": "application/json", "Checksum": "sha256=6ca13d52ca70c883e0f0bb101e425a89e8624de51db2d2392593af6a84118090" }
