    Create A BTC Multisig Wallet

    Create a new BTC multisig wallet.

    import requests

requests.post(
    url="https://testnet.sectoken.io/wallet/btc/create/", 
    headers={"HMAC": "your token"}, 
    json={
        "pubkeys": [
            "020fcbb44ff4a1f423499b1eb9b2bd9cbe467934467d4857451c7eab7f8cb2885b", 
            "038f2d1612ae08a9595fbe1d30c9b13e748a4ae55c06d5d3fa5c0b2e59712d3b6b"
        ],
        "privkeys":[
          "ZDkyM2ExMzZhYTM0OTJiNTJiMGI3NTc1N2U1NjVmODM3MzE4YWI2ZDQwNjQ4MTI0MWU5OGFhNWYwOWFjY2Q2Mg==", 
         "ZWJkMGUwZDRlNWQ0MWQyMzFiMGM2ZWNiOTY3YjQwMzExMDdmNmVhMmI4NzJlZTVjMWRmYzMyZjgzOWMyOTA5ZA==", 
        ],
        "wallet_name": "your wallet name",
    }
)

    HTTP Request

    POST https://testnet.sectoken.io/wallet/btc/create/

    Request Parameters

    Parameter Name Type Required Description
    pubkeys array Yes Public key list
    privkeys array Optional Private key list. Must be base64 encoded.
    wallet_name string Yes wallet name

    Response Parameters

    Parameter Name Type Description
    code integer Status code.If success ,returns 200. If not，check the Error Codes
    data object Info about the newly created wallet

    data Parameters

    Parameter Name Type Description
    wallet_id string wallet id
    activate_code string Activate code
    server_pubkey string BIP39 HD public key

    Activate Wallet

    Activate the wallet with 'wallet_id' and 'activate_code', both of the which are coming from the step "Create A BTC Multisig Wallet".

    import requests

requests.post(
    url="https://testnet.sectoken.io/wallet/btc/activate/",
    headers={"HMAC": "your token"},
    json={
        "wallet_id": "your wallet id",
        "activate_code": "your activate code",
    } 
)

    curl "https://testnet.sectoken.io/wallet/btc/activate/" \
    -H "HMAC:your token" \
    -H "Content-Type:application/json" \
    -X POST \
    -d '{"wallet_id": "your wallet id", "activate_code": "your activate code"}'

    Response Samples

    {
    "code": 200,
    "data": {
        "address": "2NA8Mxe7Ex6B5rk1cV7Xoijkwp7AG6m4nWc"
    }
}

    Parameter Name Type Required Description
    wallet_id string Yes wallet id
    activate_code string Yes Activate code

    Response Parameters

    Parameter Name Type Description
    code integer Status code. If success ,returns 200. If not，check the Error Codes
    data object Response data

    data Parameters

    Parameter Name Type Description
    address string BTC address of the wallet

    HTTP Request

    GET https://testnet.sectoken.io/wallet/btc/detail/

    Request Parameters

    Parameter Name Type Required Description
    wallet_id string Yes wallet id

    Response Parameters

    Parameter Name Type Description
    code integer Status code. If success ,returns 200. If not，check the Error Codes

    data Parameters

    freeze_overtime integer Wallet freeze deadline. 0 for not frozen.

    Add Wallet WebHook

    Add a webhook that will result in an HTTP callback at the specified URL from Sectoken when events are triggered.

    There is a limit of 10 webhooks per wallet.

    import requests

requests.post(
    url="https://testnet.sectoken.io/wallet/btc/webhook_add/", 
    headers={"HMAC": "your token"}, 
    json={
        "wallet_id": "your wallet id",
        "url": "https://your.server.com/webhook",
        "label": "test callback",
        "type": "receipt",
    }
)

    curl "https://testnet.sectoken.io/wallet/btc/webhook_add/" \
    -H "HMAC:your token" \
    -H "Content-Type:application/json" \
    -X POST \
    -d  '{"wallet_id": "your wallet id", "url":"https://your.server.com/webhook", "label":"test callback", "type":"receipt"}'

    Response Samples

    {
    "code": 200,
    "data": {
        "callbackid": "da0e3f3530cb7f63695d8eaad351fdc1" 
    }
}

    HTTP Request

    POST https://testnet.sectoken.io/wallet/btc/webhook_add/

    Request Parameters

    Parameter Name Type Required Description
    wallet_id string Yes Wallet id
    url string Yes URL to fire the webhook to
    type string Yes receipt or payment
    label string Optional Label of the new webhook

    Response Parameters

    Parameter Name Type Description
    code integer Status code. Successful return 200. Others are error codes.
    data object Response data.

    Data Parameters

    Parameter Name Type Description

    Transaction Callback

    After making a transfer， Sectoken will make a POST request to a url specified by the user. A service designed by yourself should be bind to the url to deal with the request. Need to add the wallet Webhook first.

    The content-type header of the request is application/json.

    curl "http://your.server.com/diy/url" \
    -X post -H "content-type:application/json"
    -d '{"wallet_id": "6004e156344ec27cad24a8df5bbaa5ab", "txid": "46ab8299ed14ba7d57b57210b7756d80f67b3db2448fd4d1f1fb78039e5bfd6d", "height": 1431122, "output": [{"address": "2N4b5ham6n4y4gMkrN8R5sswvvmJdrGV3u7", "value": "1000000000"}, {"address": "2N1DDUWKa3kdaKaPrPX5eZEMgFHxtU2JPmG", "value": "363"}], "type": "payment", "timestamp": 1572510180, "coin_type": "BTC", "decimal": 8, "memo": "", "callbackid": "fa5dc2025e79c37b8656ef4b89619d9d"}'
