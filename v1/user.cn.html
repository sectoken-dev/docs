注册用户

通过该接口可以注册用户

curl "https://testnet.sectoken.io/api/v1/wallet/user/register/" \
    -H "Content-Type:application/json" \
    -X POST \
    -d '{
        "email": "$EMAIL",
        "password": "$PASSWORD", 
        "captcha_code": "wlar",
        "captcha_key": "30c4327d90e9d41bf0f078933a1a3f7c"
    }'

返回示例

{"code": 200}

HTTP Request

POST https://testnet.sectoken.io/api/v1/wallet/user/register/

请求参数

参数名 参数类型 必填 说明
email string 是 邮箱名称
password string 是 密码
captcha_code string 是 图片验证码
captcha_key string 是 图片验证码对应的key

返回参数

参数名 参数类型 说明
code integer 返回码，200成功，其他失败，请查看错误码

激活账户

通过邮箱获取邮件验证码,然后可以激活用户

curl "https://testnet.sectoken.io/api/v1/wallet/user/active/" \
    -X POST \ 
    -H "Content-Type:application/json" \
    -d '{"email":"$EMAIL", "activate_code":"$ACTIVATE_CODE"}'

返回示例

{"code": 200}

返回参数

添加DEV Token

通过该接口可以生成一个devtoken, 该令牌只返回一次, 应妥善保管

curl "https://testnet.sectoken.io/api/v1/wallet/user/token_add" \
    -H "Authorization: Bearer $BEARER_TOKEN" \
    -H "Content-Type:application/json" \
    -d '{
    "label": "test",
    "expire_time": 15552000,
    "ip_white": ["192.168.39/24", "222.173.28.158"],
    "scopes": ["view", "spend", "create", "admin"],
    "totp_code": "000000"
    }'

返回示例

{
    "code": 200,
    "data": 
    {
     "devtoken": "Atk32053d7edd643b762ccc5c82a62c3273", 
     "token_id": "88a100c1fef2500423e72069e499f3e7"
    }
}

HTTP Request

GET https://testnet.sectoken.io/api/v1/wallet/user/token_add/

请求参数

参数名 参数类型 必填 说明
wallet_ids array 否 允许token访问的钱包列表
label string 是 token的备注名称
expire_time integer 是 token的有效时长(单位: 秒)
scopes array 是 允许token的访问权限类型
ip_white array 否 允许访问的ip地址, 可以使用 IP簇
totp_code string 是 谷歌验证码, 当没有绑定时会要求先绑定谷歌验证码

返回参数

参数名 参数类型 说明
code integer 返回码，200成功，其他失败，请查看错误码
data object 返回内容

data 参数

参数名 参数类型 说明
devtoken string 访问令牌
token_id string token的标识ID

删除DevToken

通过该接口可以删除一个devtoken, 被删除的devtoken将无法再进行任何操作。该操作只能在页面进行操作

curl "https://testnet.sectoken.io/api/v1/wallet/user/token_opt/" \
    -H "Authorization: Bearer $BEARER_TOKEN" \
    -H "Content-Type:application/json" \
    -X POST \
    -d '{
            "wallet_id": "cf761ad1b6c5f30a1557d942cb557b07",
            "url": "$WEBHOOK_URL", 
            "label": "Receipt", 
            "type": "withdraw", 
            "coin": "user",
            "secret_key":"abc123"
    }'

返回示例

{
    "code": 200,
    "data": {
        "webhook_id": "da0e3f3530cb7f63695d8eaad351fdc1" 
    }
}

HTTP Request

POST https://testnet.sectoken.io/api/v1/wallet/user/token_opt/

请求参数

参数名 参数类型 必填 说明
token_id string 是 令牌唯一标识id
totp_code string 是 接受通知的url地址, 必须使用HTTP或者HTTPS协议

返回参数

参数名 参数类型 说明
code integer 返回码，200成功，其他失败，请查看错误码

错误码

当发生错误时返回的错误码

错误码 错误说明
4000 请求参数错误
4004 未找到请求资源
5000 未知错误, 请联系Sectoken
4201 钱包不存在
4203 钱包下没有此地址
4204 钱包余额不足
4210 钱包Webhook数目超限
4211 地址格式错误
4401 非法操作
4402 此钱包不存在
4403 钱包个数达到上限
4404 用户权限不足
4405 用户不存在
4406 用户已在钱包中
4407 用户不在钱包中
4503 目标地址不能是自己
4504 签名错误
4505 全部转出只能有一个收款地址
4506 公钥格式不正确
4507 私钥密文必须是Base64格式
4508 webhook 不存在
4509 不能提交重复签名
4510 每个用户只能签名一次
4511 非当前用户所属私钥签名
4512 无法广播已完成的交易单
4513 unspent 已经被使用
4607 公钥地址已被使用
4608 用户全部控制的钱包无法派生地址
4609 当前用户持有的公钥数量已达上限
